# 🔒 TradeCoin 보안 가이드

## API 키 관리 정책

### 🎯 **기본 원칙**
- `.env` 파일에는 **절대로 실제 API 키를 넣지 마세요**
- 각 사용자는 앱에서 **개인 API 키를 직접 입력**합니다
- 개발자는 필요시 `.env.developer` 파일을 로컬에서만 사용

### 🔧 **개발자 가이드**

#### 1. 일반 개발/테스트
```bash
# .env 파일의 기본 설정 사용 (Mock 데이터)
npm start  # 또는 python main.py
```

#### 2. 실제 API 키가 필요한 경우
```bash
# .env.developer 파일에 개인 키 입력 (Git에 커밋되지 않음)
cp .env.developer.example .env.developer
# 개인 API 키 입력 후 사용
```

### 👥 **사용자 관리**
- 사용자별 API 키는 암호화되어 데이터베이스에 저장
- 각 사용자의 개인 바이낸스 계정과 독립적으로 연결
- 키 분실시 앱에서 재설정 가능

### ⚠️ **보안 주의사항**
1. **절대 금지**: 실제 API 키를 코드에 하드코딩
2. **필수**: .env.developer 파일을 .gitignore에 추가
3. **권장**: 테스트넷 키 사용 (BINANCE_SANDBOX=true)
4. **필수**: API 키에 출금 권한 부여 금지

### 🚨 **보안 사고 대응**
1. API 키 노출 발견시 즉시 바이낸스에서 키 삭제
2. 새로운 키 생성 및 권한 재설정
3. 관련 계정 활동 모니터링

### 🔍 **감사 체크리스트**
- [ ] .env 파일에 실제 API 키 없음
- [ ] .env.developer 파일이 .gitignore에 있음
- [ ] 사용자 키가 암호화 저장됨
- [ ] 출금 권한이 비활성화됨
- [ ] IP 제한이 설정됨 (권장)
